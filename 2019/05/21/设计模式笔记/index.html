2 设计模式

2.1 工厂模式

将new操作单独封装，用于返回类的实例

案例：
jquery实例化
React.createElement()
vue异步组件

2.2 单例模式

表示全局中只有一个实例，即只能被实例化一次。用于需要全局访问数据的对象。由于js中没有private关键字修饰，因此只能通过注释提示不能通过new关键字来创建对象。

案例：
jquery中的实例$
购物车
vuex和redux中的store实例
windows操作系统中的回收站

2.3 适配器模式

旧接口格式和使用者不兼容，需要中间加个中间类或方法，使用者通过调用修改的方法，修改的方法调用旧接口，从而达到间接使用旧接口的目的。

案例：
封装旧接口
vue computed返回计算后的值

2.4 装饰器模式

装饰器是为对象或者方法添加新功能，而不改变原有结构和功能

案例：
es7中的装饰器

将类的属性值设置成只读
增强类的方法
core-decorators中提示方法是否过时的deprecate

2.5 代理模式

使用者无权访问目标对象，需要通过访问代理对象的方式间接访问

案例：
网页事件代理
jquery proxy
es6 proxy
vpn

2.6 外观模式

为子系统中的一组接口提供一个高层接口，用户不直接访问子系统，而是访问高层接口(我觉得像正向代理)
案例：
jQuery中的传递参数可以不同

2.7 观察者模式

发布订阅

案例：
网页事件绑定
promise
nodejs自定义事件

vue,react生命周期钩子

2.8 迭代器模式

顺序访问一个集合，使用者无需知道集合结构

案例：
jQuery each
ES6 Iterator

2.9 状态模式

一个对象有多个状态，将对不同状态的处理过程抽象成单独的类，而非简单实用if…else来处理
案例：
有限状态机(javascript-state-mchine)
promise

2.10 原型模式

由于new 生成对象时开销比较大，所以使用clone自己生成新的对象

2.11 组合模式

文件夹模型，如虚拟dom中的vnode对象

2.12 享元模式

共享内存，相同数据共享使用

2.13 策略模式

不同策略分开处理，避免if…else

2.14 模板方法模式

相同代码抽象成方法

2.15 职责链模式

不同的角色分开来，一步步处理
案例：
jqurey链式调用
promise中的then

2.16 命令模式

执行命令式，发布者和执行者分开

2.17 备忘录模式

随时记录一个对象的状态，随时可以恢复之前的状态，如vscode编辑器

2.18 中介者模式

多个对象之间频繁访问，可以在中间添加一个中介者对象，对象通过中介者访问其他对象

2.19 访问者模式

将数据操作和数据结构分离

2.20 解释器模式

描述语言语法如何定义，如何解释和编译
