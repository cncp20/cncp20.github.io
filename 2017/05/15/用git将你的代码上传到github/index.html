用git将你的代码上传到github

发表于 2017-05-15

github是全球最大的代码托管仓库，各大公司都会将自己的开源项目上传至此，在上面可以学习大牛们的思想，看看他们是如何编写代码的。当然你也可以将自己的代码上传上去，这是一个非常好的展示自己的平台，如果你足够牛叉，说不定会有大公司向你抛出橄榄枝。下面我就教你如何将自己的代码上传到github上。

##注册账户以及创建仓库
第一步当然就是注册啦，打开https://github.com

填写相应的信息，点击sign up for github。
注册成功之后，点击“+”后选择“new repository”新建代码仓库。

填写以下信息，点击“create repository”创建仓库。建议在本地创建README.md，该文件是对本项目的详细描述信息。

如果没有在创建时添加README，会跳转以下界面

如果添加README，会跳转以下界面

##安装git客户端
github是服务端，要想在自己电脑上使用git我们还需要一个git客户端，
windows用户请下载 https://git-for-windows.github.io/
一路next，安装成功后， 在任何文件夹下，点鼠标右键会多出一些菜单
如 Git Bash Here、Git Gui Here， 说明安装成功。

##配置git

###配置用户名和密码

git config --global user.name "你的用户名"
git config --global user.email "你注册时的邮箱地址"

在全局配置你的用户名和邮箱地址

###配置密钥
在gitbash里输入以下命令

ssh-keygen -t rsa -C "注册时的邮箱地址" ，

一路回车之后就会生成自己的公钥与私钥，然后打开所在文件夹，即上图红框显示的地址

id_rsa是私钥，id_rsa.pub是公钥，打开公钥等下要用到，如果打不开，可以先改成txt后缀，复制里面所有的内容，记得改回来。

###github添加密钥

进入github，点击头像，选择settings

最后点击add ssh key就可以把密钥添加到github上，添加好了后会跳转以下界面

可以用ssh -T git@github.com测试自己是否可以连接成功了，中间要输入一个yes，出现以下界面就说明配置好了。

##上传代码

打开你需要上传的文件夹，在空白处右键，选择git bash here。会弹出git命令面板。

###初始化

git init

文件夹下会出现一个.git的隐藏文件,如果没看到，可以去控制面板选择显示隐藏文件，就说明初始化成功了。

###将代码添加到缓存区

git add .

“.”表示将所有文件添加到缓存区，也可以写具体文件，比如git add README.md

###将修改保存到本地仓库中

git commit -m "first commit"

引号里面的内容是自定义的，一般表示这次修改了那些部分

###将代码推送到github远程服务器上

git remote add music-vue git@github.com:cncp20/music-vue.git
git push -u music-vue master

git remote add 自定义名字（我的习惯是和远程仓库同名） git@github.com:github用户名/仓库名称.git。
git push -u 自定义名字 master（代表提交到主分支）
第一次上传会要求输入用户名和密码
这样你的代码就会上传到github上

###如果修改了本地代码，再次提交可执行以下几个步骤

git add .
git commit -m "message"
git push -u 仓库名称 master

tips：如果你在创建仓库的时候让服务器生成了README的话，就要先将仓库克隆到本地，在push之前执行以下命令

git pull –rebase 远程仓库名 master

此代码保证远程仓库与本地代码同步，再去push就可以了
